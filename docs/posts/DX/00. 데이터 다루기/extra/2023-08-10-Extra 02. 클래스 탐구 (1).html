관찰1 : __str__

- 아래둘은 같은 역할을 한다. \(\to\) 즉, print(“a”)는 print("a".__str__())의 축약 버전이다.

 print("a")
a
print("a".__str__())
a

- 우리가 정의한 show함수는 다음과 같다.

def show(self):
        print('a={}'.format(self.a))

- 즉, 작성한 show함수는 단지 print문을 호출하는 함수이므로, a.__str__()의 기능을 재정의하면? print(a)의 결과도 바뀌지 않을까?

깨달음 4. __repr__

- 우리가 어떤 변수를 할당하고 실행할때 사용되는 내장 함수는 __repr__() \(\to\) representation의 약자이다.

- 그러면 __repr__도 우리가 정의할 수 있지 않을까?

- __str()__ 과 비교해보자.

a = np.arange(4).reshape(2,2)
a.__str__()
'[[0 1]\n [2 3]]'
a.__repr__()
'array([[0, 1],\n       [2, 3]])'

- print 문을 사용한 비교.

print(a.__str__())
[[0 1]
 [2 3]]
print(a.__repr__())
array([[0, 1],
       [2, 3]])

class의 적용

def __repr__(self):
    return 'a={}'.format(self.a)
class UpJump:
    def __init__(self):
        self.reset()
    def up(self):
        self.a = self.a + 1  
        print("a의 값이 1 증가합니다.")
    def jump(self,jump_size):
        self.a = self.a + jump_size      
        print("a의 값이 {} 증가합니다.".format(jump_size))
    def __str__(self):
        return 'a의 값은 {}입니다.'.format(self.a)
    def reset(self):
        self.a = 0
        print("a의 값이 0으로 초기화 되었습니다.")
    def __repr__(self):
        return 'a={}'.format(self.a)
