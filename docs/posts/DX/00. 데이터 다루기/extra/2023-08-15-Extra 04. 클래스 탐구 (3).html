다중상속 (super X)

일반적인 다중 상속

- Add 클래스 선언

class Add:
    def __init__(self,value):
        self.value = value 
    def __add__(self,value2):
        return self.value + value2

- 초기화

a = Add(2)
a.value
2
a + 5 
7

- Mul 클래스 선언

class Mul:
    def __init__(self,value):
        self.value = value 
    def __mul__(self,value2):
        return self.value * value2
a = Mul(5)
a.value
5

- 더하기는 위에서 정의한적 없음

- 곱하기 수행

10

- 위 2개의 클래스를 상속

class am(Add,Mul) :
      pass

믹스인 클래스(\(\star\star\star\))

class Init:

class Add(Init):
    def __add__(self,value2):

다중상속 (super O)

super를 쓰지 않는 좋지 못한 예시

class Init:
    def __init__(self,value):
        ## 여기는 Init 클래스야 
        print("Init클래스에서 정의된 __init__메소드를 실행합니다")        
        self.value = value

- 위 intit을 상속 받아서

  • 초기값 = 초기값 x 2

  • 초기값 = 초기값 + 5

를 객채 생성과 동시 에 수행하는 클래스를 각각 만듬

class a1(Init):
    def __init__(self,value):
        Init.__init__(self,value)
        self.value = self.value * 2

싫은 이유

  1. 코드가 지저분함

  2. 먼가 상속이란 단어의 진정성이 없음

super()을 활용한 좋은 예시

class Init(object):
    def __init__(self,value):
        ## 여기는 Init 클래스야 
        print("Init클래스에서 정의된 __init__메소드를 실행합니다")        
        self.value = value
        
class a1(Init):
    def __init__(self,value):
        super().__init__(value)
        self.value = self.value * 2
        
class a2(Init):
    def __init__(self,value):
        super().__init__(value)
        self.value = self.value + 5
        
class a3(a2,a1):
    def __init__(self,value):
        super().__init__(value)
a = a3(5)

super의 사용방법

- (“초기값 x 2 + 5) x 2” 를 수행해주는 클래스를 만들고 싶음.

class Init(object):
    def __init__(self,value):
        ## 여기는 Init 클래스야 
        print("Init클래스에서 정의된 __init__메소드를 실행합니다")        
        self.value = value
        
class a1(Init):
    def __init__(self,value):
        super().__init__(value)
        self.value = self.value * 2
        
class a2(Init):
    def __init__(self,value):
        super().__init__(value)
        self.value = self.value + 5
        
class a3(a2,a1): ## 파라미터 순서에 따라 mro 상속 순서가 달라짐
    def __init__(self,value):
        super().__init__(value)
        super(a2,self).__init__(self.value)

- super().__init__(value) : 상위 클래스인 init,a1,a2의 __init__를 순서대로 실행하되 중복실행하지 않는다.

- super(a2,self).__init__(self.value) : a2보다 상위 클래스인 init, a1의 __init__을 순서대로 실행하되 중복실행은 하지 않음.

a3.mro()

- 그냥 이게 더 낫지 않나?

class Init(object):
    def __init__(self,value):
        ## 여기는 Init 클래스야 
        print("Init클래스에서 정의된 __init__메소드를 실행합니다")

class a1(Init):
    def aa1(self):

        self.aa1()

- 그래도 이전 방법들을 잘 알아야한다.

  • 어떠한 클래스를 상속 받을때는 “내가 만든 클래스”가 아닐 경우가 대부분이다. 따라서 “애초부터 메소드가 겹치지 않게 클래스들을 깔끔하게 디자인을 하는것”은 불가능한 경우가 많다.

리스트의 상속

- list와 비슷하면서… 멤버들의 빈도가 계산되는 메소드를 갖는 나만의 list 클래스를 만들고 싶다.

lst = list('asdfasssdfa')
lst 
['a', 's', 'd', 'f', 'a', 's', 's', 's', 'd', 'f', 'a']

- 각 원소들의 빈도

{i : lst.count(i) for i in set(lst)}
{'d': 2, 's': 4, 'f': 2, 'a': 3}
  • lst.freq()를 수행하면 위의 결과가 나왔으면 좋겠음
class List(list) :
    def freq(self) :
        return {i : self.count(i) for i in set(self)}
lst2 = List('asdfasssdfa')
lst2
['a', 's', 'd', 'f', 'a', 's', 's', 's', 'd', 'f', 'a']
lst2.freq()
{'d': 2, 's': 4, 'f': 2, 'a': 3}
